Modelling¶

Definitely a time series problem. But we need also to suggest a driver to a nearest area which has high demand(high ride value).

That requires us to do some clustering and on each cluster we can forecast

  • clustering on whole data (KMeans or variation of KMeans)
  • Forecast using
    • ARIMA Family
    • Classical ML Regression algo (XGBoost, SVM regression, Linear Regression etc., but first need to formalize it as supervise learning problem)
    • Deep Learning (LSTM)

def create_features(df):
  #Resampling Data in region j into one hour step
  df = df.resample('1h').sum() 
  #Feature Engineering
  df['Date time'] = df.index
  df['Date time'] = pd.to_datetime(df['Date time'].dt.strftime('%Y-%m-%d %H'))
  df['hour'] = df['Date time'].dt.hour 
  df['day'] = df['Date time'].dt.day
  df['dayofweek'] = df['Date time'].dt.dayofweek  
  df['workingday'] = df.apply(lambda x: 1 if x.dayofweek < 5 else 0, axis=1)
  df['demand'] = df['demand'].fillna(0)
  return df

def sarimax_gridsearch(ts, pdq, pdqs, maxiter=5, freq='H'):
    # Run a grid search with pdq and seasonal pdq parameters and get the best BIC value
    ans = []
    for comb in pdq:
        for combs in pdqs:
            try:
                mod = sm.tsa.statespace.SARIMAX(ts, # this is your time series you will input
                                                order=comb,
                                                seasonal_order=combs,
                                                enforce_stationarity=False,
                                                enforce_invertibility=False,
                                                freq=freq)

                output = mod.fit(maxiter=maxiter) 
                ans.append([comb, combs, output.bic])
                print('SARIMAX {} x {}24 : BIC Calculated ={}'.format(comb, combs, output.bic))
            except:
                continue
            
    # Find the parameters with minimal BIC value

    # Convert into dataframe
    ans_df = pd.DataFrame(ans, columns=['pdq', 'pdqs', 'bic'])

    # Sort and return top 5 combinations
    ans_df = ans_df.sort_values(by=['bic'],ascending=True)[0:5]
    
    return ans_df.iloc[0]['pdq'],ans_df.iloc[0]["pdqs"]

How to convey findings to Drivers?¶

In Real-time we will get driver's current location or current trip's end location.

  • We will provide this current location and current time to the model.
    • We will find neighbor clusters to driver's current trip's end location
    • We will access neighbor clusters forecast for the next hour
    • Make a decision to driver, whether to stay in same cluster or to move to near cluster if demand is high
In [37]:
def from_forecast_retrieve_demand(current_time, cluster):
  date = pd.to_datetime(current_time, format='%Y-%m-%d %H:%M:%S')
  next_timestamp = date.floor('H')+pd.Timedelta('1h')
  d = pd.read_csv('/content/drive/My Drive/HA-Bolt/Forecast/cluster_{}_predictions.csv'.format(str(cluster).zfill(2)))
  return d[d["time"]==str(next_timestamp)]["predicted_value"].values[0]

def get_next_cluster(cab_current_location, current_time):
  global regions, centroids
  
  df = pd.DataFrame()

  df['end_lat'] = [x[0] for x in centroids]
  df['end_lng'] = [x[1] for x in centroids]

  df['start_lat'] = cab_current_location["lat"]
  df['start_lng'] = cab_current_location["lng"]

  df['next_cluster'] = regions.predict(df[["end_lat", "end_lng"]])
  df['current_cluster'] = regions.predict(df[["start_lat", "start_lng"]])
  
  df['distance_eagle'] = df.apply(lambda x: distance.distance((x.start_lat, x.start_lng), (x.end_lat, x.end_lng)).km, axis=1)
  df.sort_values('distance_eagle', inplace=True)

  tmp = df.head(5).apply(lambda x: from_forecast_retrieve_demand(current_time, int(x["next_cluster"])), axis=1)
  next_cluster = tmp.idxmax()
  return next_cluster

Future Scope¶

Dataset

  • I would consider weather data, road data with U-turn, over-bridge, current traffic,under pass information
  • OSRM open source API for calculating distance between source and destination by road
  • Any planned public event or celebrations data so that people will gather and demand will be high at specific time and location

Feature creation or processing

  • Different distance measuring technique, as eagle distance is not accurate, may be look into OSRM routes or manhattan distance
  • Adding year, quarter, Holiday information if longer period of data is available

Modelling

  • Deep-learning based appraoches like LSTM
  • AWS DeepAR to get benchmarking results
  • Classical ML regression algorithms (SVR Regression, XGBoost etc.,)

Deployment¶

  • As we mention in the section How to convey findings to drivers? we can expose get_next_cluster function to a REST API service and from driver side we will get his location and time and based on that we can provide forecast for next hour.
  • Batch mode v/s online mode: In batch mode we forecast for fix length and then use that forecast without rerunning the model, In online mode, once we get actual demand we will use it to re-run the model and make forecast better

validate your solution for live operations¶

A/B Testing for effectiveness of our approach

  • Find similar cluster, assign one as a control group and other as a treatment group. For control group we will not apply our solution and for treatment group we will recommend drivers towards high demand clusters

  • Run it for a week or fortnight and analyze the results using hypothesis testing

H0: Revenue is same in both cluster H1: In treatment group revenue is higher

One-tailed test case.
